# Prototype v0.1 仕様

## 基本方針

Prototype v0.1は、スマホ縦持ちのブラウザゲームとして実装する。

プレイヤーはブロックを直接操作しない。プレイヤーは在庫からブロックを選んでAIへ供給し、SafeAIが盤面上の最善配置を自動で選ぶ。

## 画面・操作

- 画面方向はスマホ縦持ちを優先する
- 盤面、在庫、管理力、スコア、操作ボタンが縦画面内で確認できること
- プレイヤー操作はブロック供給と緊急発注を中心にする
- ブロックの左右移動、回転、落下操作はプレイヤーに行わせない

## 盤面

- 横10マス×縦20マス
- 上方向を高い位置、下方向を低い位置として扱う
- ブロックが盤面上部にはみ出す、または配置不能になった場合はゲームオーバー
- クリア判定では盤面の最高到達高さを使用する

## ブロック

供給されるブロックは以下の7種類とする。

- I
- O
- T
- L
- J
- S
- Z

各ブロックは一般的なテトリス形状を使用する。

## 在庫

プレイヤーは在庫からブロックを1つ選び、SafeAIに供給する。

初期在庫は以下とする。

| 種類 | 初期在庫 | 最大在庫 |
| --- | ---: | ---: |
| I | 2 | 3 |
| O | 4 | 5 |
| T | 3 | 5 |
| L | 4 | 5 |
| J | 4 | 5 |
| S | 4 | 5 |
| Z | 4 | 5 |

- 在庫が0のブロックは通常供給できない
- 供給したブロックは在庫を1消費する
- 在庫は最大在庫を超えない

## 自動補充

- ブロックを5個配置するごとに、自動補充が発生する
- 1回の自動補充では合計3個を補充する
- 在庫充足率が低い種類を優先して補充する
- 在庫充足率は「現在在庫 ÷ 最大在庫」で判定する
- 補充後も各ブロックの最大在庫を超えない

## 管理力

管理力はプレイヤーの供給判断を評価する値とする。

- 管理力の最大値は100
- 直前に供給したブロックと異なるブロックを連続供給した場合、管理力+15
- 直前に供給したブロックと同じブロックを連続供給した場合、管理力+0
- 管理力は100を超えない
- ゲーム開始時の初回供給は連続判定の対象外とする

## 緊急発注

- 管理力が100のとき、在庫0のブロックを1回だけ緊急発注できる
- 緊急発注は選択した在庫0ブロックを即時に1個供給可能にする
- 緊急発注を使ったら管理力を消費する
- Prototype v0.1では、緊急発注の使用後に管理力を0へ戻す
- 在庫が1以上あるブロックには緊急発注できない

## SafeAI

SafeAIは供給されたブロックについて、すべての回転と配置位置を評価し、最善配置を選ぶ。

評価に使用する要素は以下とする。

- 穴の少なさ
- 盤面高さの低さ
- 凹凸の少なさ
- ライン消去の有無と本数
- ゲームオーバー危険度の低さ

SafeAIは短期的なライン消去だけでなく、盤面を安全に保つことを優先する。

## スコア

ライン消去時のスコアは以下とする。

| 消去ライン数 | スコア |
| ---: | ---: |
| 1ライン | 100 |
| 2ライン | 300 |
| 3ライン | 500 |
| 4ライン | 800 |

ラインを消去しない配置ではスコアを加算しない。

## ステージ

- ステージ名：新人管理人研修①
- 目標スコア：5000点

## クリア条件

以下を両方満たしたとき、ステージクリアとする。

- スコアが5000点以上
- 目標達成後、盤面高さが6段以下

スコアが5000点に到達しても、盤面高さが7段以上の場合はクリアにならない。

## ゲームオーバー条件

- SafeAIが供給ブロックを配置できない
- ブロック配置後に盤面上部を超える
- その他、10×20盤面内でゲーム継続不能になった場合

## 管理レポート

ゲーム終了時に管理レポートを表示する。

管理レポートには以下を含める。

- ステージ名
- クリアまたはゲームオーバー
- 最終スコア
- 配置したブロック数
- ライン消去数
- 緊急発注の使用回数
- 最終管理力
- 最終在庫
- 最高到達高さ

Prototype v0.1では、管理レポートはプレイ結果確認用の簡易表示とする。
